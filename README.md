# 🚀 Plataforma de Estudos Dev

A **Plataforma de Estudos Dev** é um ecossistema colaborativo desenhado para transformar o aprendizado teórico em experiência prática. Criado dentro do contexto de uma Software House simulada, o projeto visa conectar desenvolvedores de diferentes níveis para a construção de uma base de conhecimento viva e comunitária.

## 🎯 O Objetivo
Nossa missão é romper a barreira entre o estudo acadêmico e o mercado de trabalho. Através da colaboração no GitHub e metodologias ágeis, focamos em:
* **Prática Real:** Desenvolvimento de funcionalidades ponta a ponta.
* **Mentoria:** Membros experientes auxiliando iniciantes em um fluxo de *Code Review*.
* **Portfólio Coletivo:** Projetos reais que demonstram capacidade de trabalho em equipe.

## ✨ Funcionalidades Principais
* **Gestão de Perfil:** Criação de conta e personalização de trilhas.
* **Trilhas de Estudo:** Marcação de progresso em tecnologias específicas.
* **Hub de Conhecimento:** Compartilhamento de links, artigos e repositórios úteis.
* **Showcase de Projetos:** Espaço para postar e receber feedback sobre projetos pessoais.
* **Comunidade Ativa:** Sistema de comentários e fórum para tirar dúvidas técnicas.

## 🛠️ Stack Tecnológica
Selecionamos tecnologias modernas que dominam o mercado atual:
* **Frontend:** [React](https://reactjs.org/) / [Next.js](https://nextjs.org/) (Foco em performance e SEO)
* **Backend:** [Node.js](https://nodejs.org/) com [TypeScript](https://www.typescriptlang.org/) (Tipagem forte e segurança)
* **Banco de Dados:** [PostgreSQL](https://www.postgresql.org/) via [Supabase](https://supabase.com/) (Escalabilidade e facilidade de autenticação)
* **Qualidade:** Jest para TDD e princípios de Clean Code.

## 🤝 Como Contribuir
Este é um projeto **Open Source** para os membros da nossa comunidade. 
1. Faça um **Fork** do projeto.
2. Crie uma **Branch** para sua feature (`git checkout -b feature/nome-da-feature`).
3. Siga os padrões de commit estabelecidos.
4. Abra um **Pull Request** para revisão do Tech Lead.
